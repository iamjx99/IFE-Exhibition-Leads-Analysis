# IFE 展會買家需求分析專案
# 


輸出簡易的買家需求整理與優先度列表
## 專案背景與動機
我於倫敦 IFE（International Food & Drink Event）協助台灣食品包材製造商的展位營運，負責與來自不同國家的買家接觸、蒐集需求、紀錄 leads，並在三天的展覽中深度了解：
- 不同買家的採購需求（MOQ、材質、永續、交期、應用產業）
- 各國市場偏好
- 包材供應鏈的標準流程
- B2B 採購決策如何形成

在現場我觀察到一個痛點：展會蒐集的資料零散、不結構化，展後追蹤容易錯失商機。

因此我希望將資料分析能力應用於真實的展會情境，建立一套：
- 結構化的買家資料
- SQL 的市場與需求分析
- Python 的 Lead Scoring 模型
- 供業務／PM 用於展後追蹤的優先名單
- 
因為涉及公司與客戶資料，本專案將所有資料完全模擬產生，但分析邏輯與流程比照真實業界情境設計。
